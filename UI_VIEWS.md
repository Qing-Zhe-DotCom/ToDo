# ToDo 主视图与交互说明

本文档描述的是当前仓库中主视图与交互层的真实状态；未来路线会单列说明，但不会把目标态当成已完成功能。

如果你想先看整体架构，请看 [ARCHITECTURE.md](./ARCHITECTURE.md)。  
如果你只关心用户怎么用，请看 [USER.md](./USER.md)。

## 1. 当前 UI 总体结构

当前 UI 不是“每个页面一个 FXML + Controller + ViewModel”的最终形态，而是：

- 一个 FXML shell
- 一个主控制器
- 多个 Java 自定义视图
- 一个右侧详情面板

当前入口链路：

```text
MainApp
  -> main-shell.fxml
  -> MainController
  -> ScheduleListView / TimelineView / HeatmapView / FlowchartView
  -> InfoPanelView
```

关键文件：

- [`todo/src/main/resources/com/example/ui/main-shell.fxml`](./todo/src/main/resources/com/example/ui/main-shell.fxml)
- [`todo/src/main/java/com/example/controller/MainController.java`](./todo/src/main/java/com/example/controller/MainController.java)

## 2. 主壳层与导航

### FXML shell

当前 FXML shell 非常轻量，核心就是一个 `BorderPane` 容器，控制器为 `MainController`。

它负责承载：

- 左侧边栏
- 中央主视图
- 右侧详情面板

### `MainController` 的当前职责

`MainController` 当前仍承担较多编排职责，包括：

- 创建左侧边栏
- 创建右侧详情面板
- 初始化主视图
- 在不同主视图之间切换
- 打开设置弹窗
- 打开登录占位提示
- 打开新增 / 编辑弹窗
- 协调完成状态跨视图同步
- 在设置详情页显示当前应用版本

因此当前它是一个典型的过渡态 shell controller，而不是最终精简后的页面控制器。

## 3. `ScheduleListView`

### 定位

当前最完整的主工作视图，承担日常任务录入、筛选、排序、状态切换等主要操作。

### 当前真实能力

- 显示任务列表
- 搜索结果承接
- 筛选：
  - 全部
  - 未完成
  - 已完成
  - 高优先级
  - 即将到期
- 排序与近期待办控制
- 完成状态切换
- 打开详情面板
- 打开编辑弹窗

### 交互特点

- 已完成区域支持折叠
- 状态切换带有卡片收缩、重排、落位等视觉反馈
- 当前列表不是完全静态表格，而是偏工作台式交互界面

## 4. `TimelineView`

### 定位

横向时间轴视图，用于把带日期信息的日程按时间范围铺在同一条轴上展示。

### 当前真实能力

- 顶部提供：
  - 开始日期
  - 结束日期
  - 重置视角
- 鼠标滚轮控制横向滚动
- 会尽量自动滚到今天附近
- 点击卡片：打开详情面板
- 双击卡片：打开编辑弹窗
- 点击状态控件：切换完成状态

### 当前真实分组规则

当前时间轴不是“日 / 周 / 月 / 全部”切换模式。  
它会先按持续天数把日程分为：

- 短期日程：`< 7 天`
- 中期日程：`7 - 35 天`
- 长期日程：`> 35 天`

然后在同一张横向时间轴上分组绘制。

### 当前接入点

- 全局日程卡片样式系统
- 完成状态协同机制
- 主控制器选中态与详情面板联动

## 5. `HeatmapView`

### 定位

从“完成情况”和“时间覆盖”两个维度帮助用户回顾任务分布。

### 当前真实能力

- 支持：
  - 周视图
  - 月视图
  - 年视图
- 顶部提供：
  - 周 / 月 / 年切换
  - 上一周期
  - 回到今天
  - 下一周期
- 点击热力图单元格后：
  - 选中日期
  - 下方面板显示覆盖该日期的任务卡片
- 下方任务卡片支持：
  - 查看
  - 编辑
  - 切换完成状态

### 重要说明

热力图颜色深浅看的是“这一天完成了多少任务”，  
而下方面板显示的是“哪些任务覆盖了这一天”，两者维度并不完全相同。

### 当前附加交互

- 已完成归档区视觉反馈
- 与完成状态协同体系联动

## 6. `InfoPanelView`

### 定位

右侧详情面板，用于查看并操作当前选中的日程。

### 当前真实显示内容

- 标题
- 当前状态
- 开始 / 截止日期
- 优先级
- 分类与标签
- 提醒时间
- 描述

### 当前操作

- 编辑日程
- 删除日程
- 完成 / 标记未完成

### 关闭方式

- 点击详情面板外空白区域
- 按 `Esc`

## 7. `ScheduleDialog`

### 定位

统一的新建 / 编辑弹窗。

### 当前可编辑字段

- 名称
- 描述
- 开始日期
- 截止日期
- 优先级
- 分类
- 标签
- 颜色
- 提醒开关
- 提醒日期
- 提醒时间

### 当前校验重点

- 名称不能为空
- 截止日期不能为空
- 开始日期不能晚于截止日期

### 与数据模型的关系

当前 UI 已经在使用开始 / 截止相关时间字段，但背后的持久化模型仍处于过渡态：

- `start_at` / `due_at` 已进入 SQLite schema
- 但 UI 还没有全面切换到更完整的分钟级时间语义模型

## 8. `FlowchartView`

### 当前状态

`FlowchartView` 目前仍是占位页。  
代码里明确写着“日程流程图功能正在开发中...”，没有真正的流程图节点、连线或布局逻辑。

## 9. 设置中心

设置入口由 `MainController` 打开。

### 当前页面

- 详情
- 主题
- 样式

### 当前真实能力

- 查看当前应用版本
- 查看当前主题
- 查看当前卡片样式
- 选择内置主题
- 导入外部 CSS 主题
- 切换全局日程卡片样式

### 当前影响范围

卡片样式会同时作用于：

- 日程管理
- 时间轴
- 热力图

### 当前版本显示现实

设置详情页中的应用版本不是写死文案，而是当前构建出的应用版本。  
它来自 `pom.xml -> application-defaults.properties -> ConfigurationLoader -> AppProperties -> MainController` 这条链路。

## 10. 登录入口

当前登录按钮不是实际账号系统入口。  
点击后会弹出信息框：“登录功能开发中...”。

## 11. 跨视图共享机制

### 主题与样式共享

- 由 `ThemeService` 和 `JavaPreferencesStore` 管理
- 各主视图共享统一主题和卡片样式

### 应用版本显示

- 由配置层读取构建版本
- 设置详情页显示当前应用版本

### 完成状态协同

- 由 `ScheduleCompletionCoordinator` 协调
- 列表、时间轴、热力图、详情面板都会响应完成状态变更

### 选中态与详情面板

- 主控制器负责维护当前选中对象
- 主视图点击日程后会驱动右侧详情面板同步更新

## 12. 当前 UI 的真实形态总结

当前 UI 已经不是最早那种“所有逻辑堆在一个界面里的原型”，但也还不是最终的分屏幕 MVVM 体系。

更准确的描述是：

- 已有：
  - FXML shell
  - 主导航
  - 多个可用主视图
  - 跨视图完成状态协同
  - 统一主题和卡片样式系统
  - 设置详情页应用版本显示
- 仍未完成：
  - 登录
  - 流程图
  - 更细粒度的屏幕级 Controller / ViewModel 结构

## 13. 已确定的后续路线

后续 UI 层已确定的演进方向包括：

- 本地 SQLite 化完成后，进一步简化错误状态与启动体验
- 数据模型继续升级后，支持更完整的时间语义与更丰富的详情展示
- 逐步从“shell controller + Java 视图”演进到更细粒度的界面分层

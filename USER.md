# ToDo 用户与交互说明

本文档描述的是当前仓库中源码运行版本的真实交互状态；未来已经确定的路线会单独说明，但不会把未来目标写成当前已实现能力。

如果你想先了解项目全貌，请看 [README.md](./README.md)。  
如果你准备继续维护，请看 [DEV.md](./DEV.md)。  
如果你想了解当前技术事实，请看 [ARCHITECTURE.md](./ARCHITECTURE.md)。

## 1. 当前这是什么版本

当前仓库里的 ToDo 不是正式安装包，也不是已发布桌面成品，而是一个需要从源码运行的 JavaFX 桌面应用。

你当前可以实际使用的能力包括：

- 日程列表管理
- 时间轴视图
- 热力图视图
- 右侧详情面板
- 搜索、搜索历史与搜索建议
- 设置中心
- 主题、图标包、语言与字体切换
- 设置详情页中的应用版本显示
- 待办数量驱动的窗口标题 / 任务栏角标更新

当前仍未完成或仍是占位的部分：

- 日程流程图页面
- 登录功能
- 正式安装包 / 发布包
- 云同步与账号体系

## 2. 启动前提与启动方式

### 必备软件

- Java 21
- Maven

默认本地运行不要求你先安装或启动 MySQL。

### 当前数据库事实

当前默认运行已经切到 SQLite，本地配置默认来自 [`todo/src/main/resources/application-defaults.properties`](./todo/src/main/resources/application-defaults.properties)。

默认情况下：

- 应用优先使用 SQLite 本地文件库
- Windows 下默认数据目录通常是 `%APPDATA%\ToDo\`
- 默认 SQLite 文件通常是 `%APPDATA%\ToDo\todo.sqlite`

只有在下面这些场景里，你才需要关心 MySQL：

- 你手动把 `TODO_DB_MODE` 或配置文件中的 `todo.db.mode` 改成了 `mysql`
- 你在做兼容验证或手动迁移场景

### 如何启动当前版本

推荐在仓库根目录打开 PowerShell 后执行：

```powershell
cd .\todo

$env:JAVA_HOME='C:\Program Files\Microsoft\jdk-21.0.10.7-hotspot'
$env:JAVA21_HOME=$env:JAVA_HOME
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"

mvn -f .\pom.xml clean javafx:run
```

如果你只想先确认能否编译：

```powershell
cd .\todo
mvn -f .\pom.xml clean compile
```

如果你只想跑测试：

```powershell
cd .\todo
mvn -f .\pom.xml test
```

## 3. 主界面布局

程序启动后，界面主要分为三部分：

- 左侧边栏
- 中央主内容区
- 右侧详情面板

当前入口链路是：

```text
MainApp
  -> main-shell.fxml
  -> MainController
  -> ScheduleListView / TimelineView / HeatmapView / FlowchartView
  -> InfoPanelView
```

### 左侧边栏

左侧边栏负责：

- 切换主视图
- 搜索日程
- 展开“更多功能”区域

当前可见主入口：

- 日程管理
- 日程时间轴
- 日程热力图
- 日程流程图

底部“更多功能”展开后可见：

- 新建日程
- 登录
- 设置
- 退出

### 中央主内容区

中央区域会显示当前主视图：

- 日程列表
- 时间轴
- 热力图
- 流程图占位页

### 右侧详情面板

选中某条日程后，右侧会打开详情面板。  
它支持查看详细信息、编辑、删除和切换完成状态。

关闭方式：

- 点击详情面板外空白区域
- 按 `Esc`

## 4. 搜索与导航

左侧搜索框支持：

- 输入关键字后按回车执行搜索
- 使用搜索历史
- 基于标题、标签、分类的搜索建议

当前搜索规则：

- 搜索结果会切换到日程列表页承接展示
- 只在活跃日程中搜索
- 搜索命中主要来自名称、描述、备注、分类和标签
- 清空搜索框后会退出搜索状态并恢复普通列表

## 5. 日程列表视图

### 定位

当前最完整的主工作区，承担日常任务录入、筛选、排序、状态切换等主要操作。

### 当前真实能力

- 查看当前日程
- 快速新增日程
- 打开新建 / 编辑弹窗
- 打开右侧详情面板
- 通过筛选缩小结果范围
- 切换完成状态

### 当前可见筛选与排序要点

当前代码里至少存在这些筛选概念：

- 全部
- 未完成
- 已完成
- 高优先级
- 即将到期

列表也保留了“近期待办工作台”的控制逻辑，因此默认列表并不是无限完整清单，而更像一个近期使用视图。

### 交互特点

- 已完成区域支持折叠
- 状态切换带有卡片收缩、重排、落位等视觉反馈
- 搜索结果会直接在列表页内展示，而不是跳到独立搜索页

## 6. 时间轴视图

### 定位

横向时间轴视图，用于把带日期信息的日程按时间范围铺在同一条轴上展示。

### 当前真实能力

- 顶部提供：
  - 开始日期
  - 结束日期
  - 重置视角
- 视图会尽量自动滚到今天附近
- 点击卡片：打开详情面板
- 双击卡片：打开编辑弹窗
- 点击状态控件：切换完成状态

### 缩放与移动

- 鼠标滚轮可用于横向移动
- 配合设置中的滚轮修饰键，可用于时间轴缩放
- 时间轴缩放快捷键可在设置中心中自定义

### 当前真实分组规则

当前时间轴不是“日 / 周 / 月 / 全部”切换模式。  
它会先按持续天数把日程分为：

- 短期日程：`< 7 天`
- 中期日程：`7 - 35 天`
- 长期日程：`> 35 天`

然后在同一张横向时间轴上分组绘制。

## 7. 热力图视图

### 定位

从“完成情况”和“时间覆盖”两个维度帮助用户回顾任务分布。

### 当前真实能力

- 支持：
  - 月视图
  - 年视图
- 顶部提供：
  - 月 / 年切换
  - 上一周期
  - 回到今天
  - 下一周期
- 热力图单元格支持 tooltip
- 点击某一天后，右侧 / 下方联动区域会显示覆盖该日期的任务卡片
- 任务卡片支持查看、编辑和切换完成状态

### 重要说明

热力图颜色深浅看的是“这一天完成了多少任务”，  
而联动任务列表显示的是“哪些任务覆盖了这一天”，两者维度并不完全相同。

### 附加交互

- 已完成归档区有视觉反馈
- 与列表、时间轴、详情面板共享完成状态协同机制

## 8. 右侧详情面板

### 定位

右侧详情面板用于查看并操作当前选中的日程。

### 当前真实显示内容

- 标题
- 当前状态
- 开始 / 截止日期
- 优先级
- 分类与标签
- 提醒时间
- 描述

### 当前操作

- 编辑日程
- 删除日程
- 完成 / 标记未完成

该面板与列表、时间轴、热力图共享选中态和完成状态同步逻辑。

## 9. 新建 / 编辑弹窗

### 定位

当前版本使用同一个弹窗处理“新建”和“编辑”。

### 当前可填写字段

- 名称
- 描述
- 开始日期
- 截止日期
- 优先级
- 分类
- 标签
- 颜色标记
- 提醒开关
- 提醒日期
- 提醒时间

### 当前校验要点

- 名称不能为空
- 截止日期不能为空
- 如果填写了开始日期，则开始日期不能晚于截止日期

### 当前真实模型关系

当前界面已经围绕开始 / 截止相关时间字段工作，但整体时间模型仍在演进中：

- 应用服务与默认持久化链路已经切到 `ScheduleItem`
- 现有 UI 仍通过 `Schedule` 做兼容桥接
- 当前还不是完全统一的分钟级开始 / 结束 / 截止 / 完成时间模型

## 10. 设置中心

设置中心当前不是简单占位弹窗，而是完整的多页设置界面，主要分为 3 个页签：

- 详情
- 个性化
- 数据

### 10.1 详情页

当前可以查看或调整：

- 应用版本
- 当前主题、图标、语言、字体概览
- 语言选择
- 字体粗细
- 时间轴缩放的滚轮修饰键
- 时间轴缩放快捷键
- Labs 开关

切换语言时，程序会提示你重启以完整应用更改。

### 10.2 个性化页

当前可以调整：

- 主题家族
- 经典主题调色板
- 图标是否与主题绑定
- 图标包选择
- 自定义时间文本输入开关

### 10.3 数据页

当前可以查看或调整：

- 自定义任务选项
- 自定义标签选项
- 标签逗号拆分开关
- 已删除任务列表
- 回收站中的恢复 / 彻底删除操作

## 11. 登录与流程图当前状态

### 登录入口

当前“登录”按钮只会弹出信息提示，属于占位入口，不包含账号体系。

### 流程图页面

`FlowchartView` 当前仍是占位页。  
它会显示标题、时钟和开发中提示，但不提供实际流程图能力。

## 12. 常见问题

### 12.1 为什么默认启动不需要 MySQL

因为当前默认运行模式已经切到 SQLite，本机不需要先启动 MySQL。

### 12.2 为什么流程图页和登录还不能用

因为它们仍处于占位状态，当前仓库尚未落地账号体系和流程图功能。

### 12.3 为什么切换语言后有时要重启

因为当前语言偏好会被保存，但界面资源与字体应用在部分场景下仍需要重启才能完全刷新。

### 12.4 为什么列表、时间轴、热力图之间能同步完成状态

因为当前版本已经接入完成状态协同机制，列表、时间轴、热力图和详情面板不再各自维护完全独立的状态。

## 13. 当前事实与后续路线

当前最准确的描述不是“产品已经完成”，而是：

> 一个可运行、可维护、功能上已有主体工作流，但仍处于过渡重构阶段的 JavaFX 桌面应用。

已经明确但尚未全部落地的方向包括：

- 更完整的 `ScheduleItem` 数据模型
- 更细粒度的 UI 分层
- 云同步与账号体系
- 正式安装包与发布体系
